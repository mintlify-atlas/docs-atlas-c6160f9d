The secrets building block lets your application read credentials and other sensitive values from external secret stores without directly importing cloud provider SDKs or managing authentication. Dapr handles authentication to the secret store for you.

<CardGroup cols={2}>
  <Card title="Unified API" icon="plug">
    One API call regardless of whether secrets live in Azure Key Vault, AWS Secrets Manager, HashiCorp Vault, or Kubernetes.
  </Card>
  <Card title="Zero SDK changes" icon="code">
    Switch secret store providers by changing a component YAML file — no application code changes required.
  </Card>
  <Card title="Access policies" icon="shield">
    Restrict which secrets each application can read using Dapr's secret scoping.
  </Card>
  <Card title="Resiliency built-in" icon="rotate">
    Dapr automatically retries secret store calls according to your configured resiliency policy.
  </Card>

## HTTP API

All secret requests go through your local Dapr sidecar at `http://localhost:3500`.

### Get a single secret

```bash
GET /v1.0/secrets/{secretStoreName}/{key}
```

```bash
curl http://localhost:3500/v1.0/secrets/my-secret-store/db-password
```

Response:

```json
{
  "db-password": "s3cr3t!"
}
```

<Info>
  The response is a map from secret name to value. Multi-value secrets (where one key holds multiple fields) return multiple entries in the map.
</Info>

<ParamField path="secretStoreName" type="string" required>
  The name of the configured secret store component.
</ParamField>

<ParamField path="key" type="string" required>
  The name of the secret to retrieve.
</ParamField>

### Version and stage metadata

Pass metadata as query parameters to retrieve a specific version or stage:

```bash
# AWS Secrets Manager: get a specific version
GET /v1.0/secrets/aws-secrets/my-key?metadata.version_id=EXAMPLE1-90ab-cdef-fedc-ba987EXAMPLE

# AWS Secrets Manager: get by stage label
GET /v1.0/secrets/aws-secrets/my-key?metadata.version_stage=AWSCURRENT

# Azure Key Vault: get a specific version
GET /v1.0/secrets/azure-keyvault/my-key?metadata.version=abc123
```

### Get all secrets (bulk)

```bash
GET /v1.0/secrets/{secretStoreName}/bulk
```

Returns all secrets from the store that the application is allowed to access:

```bash
curl http://localhost:3500/v1.0/secrets/my-secret-store/bulk
```

```json
{
  "db-password": {
    "db-password": "s3cr3t!"
  },
  "api-key": {
    "api-key": "ak_live_abc123"
  }
}
```

<Warning>
  Bulk retrieval fetches all secrets in the store. Apply secret scoping policies to limit what each application can read.
</Warning>

## Supported secret stores

## Component configuration

### Azure Key Vault

```yaml
apiVersion: dapr.io/v1alpha1
kind: Component
metadata:
  name: azure-keyvault
  namespace: default
spec:
  type: secretstores.azure.keyvault
  version: v1
  metadata:
    - name: vaultName
      value: "my-vault"
    - name: azureTenantId
      value: "your-tenant-id"
    - name: azureClientId
      value: "your-client-id"
    - name: azureClientSecret
      secretKeyRef:
        name: azure-client-secret
        key: secret
```

### HashiCorp Vault

```yaml
apiVersion: dapr.io/v1alpha1
kind: Component
metadata:
  name: hashicorp-vault
  namespace: default
spec:
  type: secretstores.hashicorp.vault
  version: v1
  metadata:
    - name: vaultAddr
      value: "https://vault.example.com:8200"
    - name: vaultToken
      secretKeyRef:
        name: vault-token
        key: token
```

### Kubernetes secrets

```yaml
apiVersion: dapr.io/v1alpha1
kind: Component
metadata:
  name: kubernetes-secrets
  namespace: default
spec:
  type: secretstores.kubernetes
  version: v1
```

<Note>
  The Kubernetes secret store reads from the same namespace as the Dapr sidecar. Grant the Dapr service account `get` and `list` permissions on `Secret` resources.
</Note>

## Using secrets in component definitions

Reference secrets from other Dapr components using `secretKeyRef` instead of embedding plaintext credentials:

```yaml
apiVersion: dapr.io/v1alpha1
kind: Component
metadata:
  name: postgres-statestore
spec:
  type: state.postgresql
  version: v1
  metadata:
    - name: connectionString
      secretKeyRef:
        name: db-password       # the secret key in your store
        key: connectionString   # the field within the secret (for multi-value secrets)
  secretStore: azure-keyvault   # name of your secret store component
```

## Scoping secret access

By default, an application can read any secret in a configured store. Use a `Configuration` resource to restrict access:

```yaml
apiVersion: dapr.io/v1alpha1
kind: Configuration
metadata:
  name: my-app-config
spec:
  secrets:
    scopes:
      - storeName: azure-keyvault
        defaultAccess: deny
        allowedSecrets:
          - db-password
          - api-key
```

<Tip>
  Set `defaultAccess: deny` and explicitly list `allowedSecrets` for production workloads. This follows the principle of least privilege and limits the blast radius if a sidecar is compromised.
</Tip>

## Example: read a database connection string from Azure Key Vault

<Steps>
  <Step title="Store the secret in Azure Key Vault">
    ```bash
    az keyvault secret set \
      --vault-name my-vault \
      --name db-connection-string \
      --value "postgresql://user:pass@db.example.com:5432/mydb"
    ```
  </Step>

  <Step title="Configure the Dapr secret store component">
    ```yaml
    apiVersion: dapr.io/v1alpha1
    kind: Component
    metadata:
      name: azure-keyvault
    spec:
      type: secretstores.azure.keyvault
      version: v1
      metadata:
        - name: vaultName
          value: "my-vault"
        - name: azureTenantId
          value: "YOUR_TENANT_ID"
        - name: azureClientId
          value: "YOUR_CLIENT_ID"
        - name: azureClientSecret
          secretKeyRef:
            name: azure-sp-secret
            key: secret
    ```
  </Step>

  <Step title="Read the secret from your application">
    <CodeGroup>

    ```python python
    import requests

    def get_db_connection_string():
        response = requests.get(
            "http://localhost:3500/v1.0/secrets/azure-keyvault/db-connection-string"
        )
        response.raise_for_status()
        data = response.json()
        return data["db-connection-string"]
    ```

    ```javascript node.js
    const fetch = require('node-fetch');

    async function getDbConnectionString() {
      const res = await fetch(
        'http://localhost:3500/v1.0/secrets/azure-keyvault/db-connection-string'
      );
      const data = await res.json();
      return data['db-connection-string'];
    }
    ```

    ```go go
    package main

    import (
        "encoding/json"
        "fmt"
        "net/http"
    )

    func getDbConnectionString() (string, error) {
        resp, err := http.Get(
            "http://localhost:3500/v1.0/secrets/azure-keyvault/db-connection-string",
        )
        if err != nil {
            return "", err
        }
        defer resp.Body.Close()
        var data map[string]string
        json.NewDecoder(resp.Body).Decode(&data)
        return data["db-connection-string"], nil
    }
    ```

    </CodeGroup>
  </Step>
</Steps>
