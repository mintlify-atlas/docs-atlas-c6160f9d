## `Dapr` service

**Full name:** `dapr.proto.runtime.v1.Dapr`

Your application calls this service on its local sidecar (`localhost:50001`).

### Service invocation

<AccordionGroup>
  <Accordion title="InvokeService — invoke a method on a remote app">
    Invokes a method on another Dapr-enabled application by app ID.

    <Warning>
      `InvokeService` is deprecated. Use proxy mode service invocation instead — send your request directly through the gRPC proxy without calling this method.
    </Warning>

    ```protobuf
    rpc InvokeService(InvokeServiceRequest) returns (common.v1.InvokeResponse);
    ```

    | Request | `InvokeServiceRequest` | Target app ID and the inner `common.v1.InvokeRequest` |

    Full method name: `/dapr.proto.runtime.v1.Dapr/InvokeService`
  </Accordion>

  <Accordion title="SubscribeTopicEventsAlpha1 — streaming topic subscription (alpha)">
    Opens a bidirectional gRPC stream. The client first sends an `InitialRequest` with the subscription details, then receives `TopicEventRequest` messages from Dapr as events arrive on the topic. The client acknowledges each event by sending a `TopicEventResponse` back over the stream.

    ```protobuf
    rpc SubscribeTopicEventsAlpha1(stream SubscribeTopicEventsRequestAlpha1)
        returns (stream SubscribeTopicEventsResponseAlpha1);
    ```

    | | Type | Description |
    |-|------|-------------|
    | Client → Dapr | `SubscribeTopicEventsRequestAlpha1` | Initial subscribe request, then per-event `TopicEventResponse` acks |
    | Dapr → Client | `SubscribeTopicEventsResponseAlpha1` | Stream of `TopicEventRequest` payloads |

    Full method name: `/dapr.proto.runtime.v1.Dapr/SubscribeTopicEventsAlpha1`

    <Tip>
      This streaming subscription is an alternative to implementing the `AppCallback.OnTopicEvent` server endpoint. Use it when you prefer a client-initiated subscription model.
    </Tip>
  </Accordion>

### Cryptography

<AccordionGroup>
  <Accordion title="EncryptAlpha1 — encrypt a data stream (alpha)">
    Opens a bidirectional stream. The client sends chunks of plaintext data along with encryption options on the first message; subsequent messages carry additional payload chunks. Dapr returns ciphertext chunks on the response stream.

    ```protobuf
    rpc EncryptAlpha1(stream EncryptRequest) returns (stream EncryptResponse);
    ```

    | | Type | Description |
    |-|------|-------------|
    | Client → Dapr (stream) | `EncryptRequest` | First message: options (`component_name`, `key_name`, `key_wrap_algorithm`, etc.) + payload chunk. Subsequent messages: payload chunks only. |
    | Dapr → Client (stream) | `EncryptResponse` | Ciphertext payload chunks with sequence numbers |

    Full method name: `/dapr.proto.runtime.v1.Dapr/EncryptAlpha1`
  </Accordion>

  <Accordion title="DecryptAlpha1 — decrypt a data stream (alpha)">
    Opens a bidirectional stream. The client sends ciphertext chunks; Dapr decrypts them and returns plaintext chunks.

    ```protobuf
    rpc DecryptAlpha1(stream DecryptRequest) returns (stream DecryptResponse);
    ```

    | | Type | Description |
    |-|------|-------------|
    | Client → Dapr (stream) | `DecryptRequest` | First message: options (`component_name`, `key_name`) + ciphertext chunk. Subsequent messages: ciphertext chunks only. |
    | Dapr → Client (stream) | `DecryptResponse` | Plaintext payload chunks with sequence numbers |

    Full method name: `/dapr.proto.runtime.v1.Dapr/DecryptAlpha1`
  </Accordion>
</AccordionGroup>

## `AppCallback` service

**Full name:** `dapr.proto.runtime.v1.AppCallback`

Dapr calls your application through this interface. Your application must implement a gRPC server that satisfies this service definition and register it before starting the Dapr sidecar.

Proto source: [dapr/proto/runtime/v1/appcallback.proto](https://github.com/dapr/dapr/tree/master/dapr/proto/runtime/v1/appcallback.proto)

<AccordionGroup>
  <Accordion title="OnInvoke — handle an inbound service invocation">
    Dapr calls this method when another service invokes your application. Your implementation receives the request and returns the response.

    ```protobuf
    rpc OnInvoke(common.v1.InvokeRequest) returns (common.v1.InvokeResponse);
    ```

  <Tab title="Go">
    ```go
    import (
        pb "github.com/dapr/dapr/pkg/proto/runtime/v1"
        "google.golang.org/grpc"
    )

    conn, _ := grpc.Dial("localhost:50001", grpc.WithInsecure())
    client := pb.NewDaprClient(conn)

    // Publish an event
    _, err := client.PublishEvent(ctx, &pb.PublishEventRequest{
        PubsubName: "pubsub",
        Topic:      "orders",
        Data:       []byte(`{"orderId":"123"}`),
        DataContentType: "application/json",
    })

    // Save state
    _, err = client.SaveState(ctx, &pb.SaveStateRequest{
        StoreName: "statestore",
        States: []*common.StateItem{
            {Key: "key1", Value: []byte("value1")},
        },
    })
    ```
