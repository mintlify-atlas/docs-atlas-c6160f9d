---
title: "Secrets API"
description: "Retrieve secrets from configured Dapr secret store components, with optional key versioning and access scoping."
---

The Secrets API retrieves secrets from pluggable secret store components (Azure Key Vault, Kubernetes secrets, HashiCorp Vault, etc.). Access to secrets can be controlled by Dapr's secrets access-control policy.

## Endpoints

| Method | Path | Version | Description |
|---|---|---|---|
| `GET` | `/v1.0/secrets/{secretStoreName}/{key}` | v1.0 | Get a single secret by key. |
| `GET` | `/v1.0/secrets/{secretStoreName}/bulk` | v1.0 | Get all secrets from the store. |

---

## Get secret

```
GET http://localhost:3500/v1.0/secrets/{secretStoreName}/{key}
```

Retrieves a secret by key from the named secret store. Many secret stores support multi-valued secrets (where a single key has multiple named values); the response is always a map.

### Path parameters

<ParamField path="secretStoreName" type="string" required>
  Name of the secret store component as declared in your Dapr component YAML.
</ParamField>

<ParamField path="key" type="string" required>
  The key (secret name) to retrieve.
</ParamField>

### Query parameters

<ParamField query="metadata.version_id" type="string">
  The version of the secret to retrieve. Supported by stores such as Azure Key Vault and AWS Secrets Manager. Omit for the latest version.
</ParamField>

<ParamField query="metadata.version_stage" type="string">
  The version stage label (e.g., `AWSCURRENT`, `AWSPREVIOUS` for AWS Secrets Manager). Component-specific.
</ParamField>

<ParamField query="metadata.*" type="string">
  Additional component-specific metadata as query parameters with the `metadata.` prefix.
</ParamField>

### Response

A JSON object mapping secret value names to their string values. For stores with single-value secrets, the map contains one entry where the key equals the secret name.

<ResponseField name="{key}" type="object">
  A map of value names to secret values.
  <Expandable title="value properties">
    <ResponseField name="{valueName}" type="string">
      The secret value. For single-value secrets, `{valueName}` is the same as the secret key.
    </ResponseField>
  </Expandable>
</ResponseField>

```json
{
  "db-password": "s3cr3tpassword!"
}
```

For multi-valued secrets (e.g., when a single key contains multiple properties):

```json
{
  "db-password": "s3cr3tpassword!",
  "db-username": "admin"
}
```

### Example

```bash
# Get the latest version of a secret
curl http://localhost:3500/v1.0/secrets/myvault/db-password

# Get a specific version
curl "http://localhost:3500/v1.0/secrets/myvault/db-password?metadata.version_id=2"
```

### Error codes

| Status | Description |
|---|---|
| `200` | Secret retrieved successfully. |
| `403` | Access denied — the secret is not allowed by the Dapr secrets access-control policy for this app. |
| `404` | Secret key not found in the store. |
| `500` | Secret store error (e.g., connection failure, authentication error). |

## Get bulk secrets

```
GET http://localhost:3500/v1.0/secrets/{secretStoreName}/bulk
```

Retrieves all secrets the application is permitted to access from the named secret store in one request.

<Warning>
  This endpoint returns all secrets the calling app is authorized to access. Apply strict Dapr secrets scoping policies to limit which secrets are returned, especially in multi-tenant environments.
</Warning>

### Path parameters

<ParamField path="secretStoreName" type="string" required>
  Name of the secret store component.
</ParamField>

### Query parameters

<ParamField query="metadata.*" type="string">
  Component-specific metadata as query parameters with the `metadata.` prefix.
</ParamField>

### Response

A JSON object where each key is a secret name and the value is a map of secret value names to their string values (the same format as the single-secret endpoint).

```json
{
  "db-password": {
    "db-password": "s3cr3tpassword!"
  },
  "api-key": {
    "api-key": "myapikey123"
  },
  "service-credentials": {
    "username": "svc-account",
    "password": "svc-pass"
  }
}
```

### Example

```bash
curl http://localhost:3500/v1.0/secrets/myvault/bulk
```

### Error codes

| Status | Description |
|---|---|
| `200` | Secrets retrieved successfully. |
| `403` | Bulk access is not allowed by the access-control policy. |
| `500` | Secret store error. |

## Secrets access control

Dapr supports scoping which secrets an application can access via `Configuration` resources. When scoping is enabled:

- Apps can only call `GetSecret` for keys explicitly allowed in the policy.
- Apps can only call `GetBulkSecret` if that operation is explicitly permitted.
- Denied requests receive `403 Forbidden`.

Configure secret scoping in your Dapr Configuration resource under `spec.secrets.scopes`.
